第一章：项目概览

OpenHuman 是什么？

OpenHuman 是一个 AI 智能体桌面平台，可以连接多个 AI 大模型（如智谱 GLM、阿里云 Qwen、火山云豆包、DeepSeek、ChatGPT、Claude、本地 Ollama 等），并统一管理 WhatsApp、Telegram、Discord、Slack 等通信渠道。你通过一个桌面应用就能和 AI 对话、管理消息、定时执行任务。

安装流程总览

下面是整个安装过程的流程图，对照这个图一步一步走就行：

1. 环境准备
→
2. 网络代理
→
3. 克隆代码
→
4. 申请 API Key
→
5. 配置环境
→
6. 编译运行

步骤 2 和 4 根据个人情况可选

不想编译？直接下载安装包

如果你只是想使用而不是开发，可以跳过步骤 1/2/3/6，直接去 GitHub Releases 下载对应平台的安装包（.dmg / .exe / .deb）。安装后参考第 5-6 章配置 AI 即可。

安装前检查清单

  • 操作系统：Windows 10+、macOS 10.15+、Ubuntu 22.04+
  • 硬盘空间：至少 10 GB（含 Chromium 框架和编译缓存）
  • 内存：至少 8 GB（推荐 16 GB）
  • 网络：能访问 GitHub 和外网（中国大陆用户需要配置代理）
  • 一个 AI 大模型的 API Key（推荐国产模型：智谱 GLM / 阿里云 Qwen / 火山云豆包 / DeepSeek，或使用本地 Ollama 免费运行）

第二章：环境准备

这一章安装 4 个基础工具。请按顺序操作，每一步都有验证命令，确认通过再往下走。

2.1 安装 Git

1下载 Git 安装包

访问 https://git-scm.com/download/win，下载 64-bit 版本并双击安装。安装过程中全部保持默认选项即可。

1安装 Git

打开终端（Terminal），执行：

xcode-select --install
1安装 Git
sudo apt update && sudo apt install -y git
验证安装
git --version

应输出类似 git version 2.45.0 的信息。

2.2 安装 Rust

为什么需要 Rust？

OpenHuman 的核心引擎用 Rust 语言编写，必须安装 Rust 编译器才能从源码构建。

1安装 Rust 工具链
# macOS / Linux
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
# Windows - 访问以下地址下载安装器
# https://www.rust-lang.org/tools/install
# 下载 rustup-init.exe 并运行

安装过程中全部按回车选择默认选项。

2刷新环境变量
# macOS / Linux（安装完后执行）
source "$HOME/.cargo/env"

Windows 用户关闭并重新打开终端即可。

验证安装
rustc --version

应输出类似 rustc 1.93.0。

2.3 安装 Node.js

版本要求：必须 >= 24.0.0

OpenHuman 要求 Node.js 24 或更高版本。如果你已有旧版本，请先升级。

1安装 Node.js（推荐使用 fnm 管理器）
# 先安装 fnm（Node 版本管理器）
# macOS / Linux
curl -fsSL https://fnm.vercel.app/install | bash
source ~/.bashrc

# Windows（PowerShell 管理员模式）
winget install Schniz.fnm

# 然后安装 Node.js 24
fnm install 24
fnm use 24

如果你不想用 fnm，也可以直接从 nodejs.org 下载安装。

验证安装
node -v

应输出 v24.x.x。

2.4 安装 pnpm

1安装 pnpm 包管理器
npm install -g pnpm@10.10.0
验证安装
pnpm -v

应输出 10.10.0。

2.5 Windows 额外步骤

仅 Windows 用户需要

macOS 和 Linux 用户可跳过本节。

1安装 Visual Studio Build Tools
  1. 访问 Visual Studio Build Tools 下载页

第三章：网络代理配置

谁需要看这一章？

中国大陆用户必看 — 访问 GitHub、下载 Rust 依赖和 Node 包可能很慢或失败。
海外用户可跳过本章。

3.1 Git 代理设置

1查看你的代理端口

打开你的代理软件（如 Clash、V2rayN），找到本地监听端口：

代理软件HTTP 端口SOCKS5 端口
Clash for Windows78907891
V2rayN1080910808
Shadowsocks10801080
2配置 Git 代理
# 把 7890 换成你代理软件的实际端口
# HTTP 代理方式
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890

# 或者用 SOCKS5 方式（二选一）
git config --global http.proxy socks5://127.0.0.1:7891
git config --global https.proxy socks5://127.0.0.1:7891
3验证代理
git config --global --get http.proxy

应输出你刚才设置的代理地址。

取消代理（代理软件关闭后执行）

git config --global --unset http.proxy
git config --global --unset https.proxy

3.2 Cargo（Rust）镜像加速

1配置 Rust 下载镜像（使用 rsproxy）
# macOS / Linux
mkdir -p ~/.cargo
cat > ~/.cargo/config.toml << 'EOF'
[source.crates-io]
replace-with = "rsproxy"

[source.rsproxy]
registry = "https://rsproxy.cn/crates.io-index"

[registries.rsproxy]
index = "https://rsproxy.cn/crates.io-index"

[net]
git-fetch-with-cli = true
EOF
# Windows PowerShell
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\.cargo"
Set-Content -Path "$env:USERPROFILE\.cargo\config.toml" -Value @"
[source.crates-io]
replace-with = "rsproxy"

[source.rsproxy]
registry = "https://rsproxy.cn/crates.io-index"

[registries.rsproxy]
index = "https://rsproxy.cn/crates.io-index"

[net]
git-fetch-with-cli = true
"@
2配置 Rustup 镜像
# macOS / Linux - 加到 ~/.bashrc 或 ~/.zshrc 末尾
export RUSTUP_DIST_SERVER="https://rsproxy.cn"
export RUSTUP_UPDATE_ROOT="https://rsproxy.cn/rustup"
# Windows PowerShell - 加到系统环境变量
[Environment]::SetEnvironmentVariable("RUSTUP_DIST_SERVER", "https://rsproxy.cn", "User")
[Environment]::SetEnvironmentVariable("RUSTUP_UPDATE_ROOT", "https://rsproxy.cn/rustup", "User")

3.3 pnpm（Node.js）镜像加速

1设置 npm 镜像源
pnpm config set registry https://registry.npmmirror.com

3.4 让所有命令走代理（通用方法）

1设置终端全局代理
# macOS / Linux - 在终端执行（每次开新终端都要执行，或加到 ~/.bashrc）
export http_proxy=http://127.0.0.1:7890
export https_proxy=http://127.0.0.1:7890
export all_proxy=socks5://127.0.0.1:7891
# Windows PowerShell - 每次开新终端都要执行
$env:http_proxy="http://127.0.0.1:7890"
$env:https_proxy="http://127.0.0.1:7890"

设置全局代理后，Git、Cargo、pnpm、curl 等所有网络请求都会走代理。

第四章：克隆代码仓库

1克隆代码（含子模块）
git clone --recurse-submodules https://github.com/tinyhumansai/openhuman.git
cd openhuman

--recurse-submodules 不能省略！项目包含 CEF（Chromium 框架）等子模块。

2如果忘记加子模块
git submodule update --init --recursive
3配置环境变量文件
cp .env.example .env
cp app/.env.example app/.env.local

暂时不需要编辑，后面会一步步配置。

4安装前端依赖
pnpm install

等待安装完成（约 1-3 分钟）。

第五章：申请大模型 API Key

OpenHuman 需要一个 AI 大模型来驱动对话和智能功能。你有 8 种选择，选一种即可：

国产大模型推荐

智谱 GLM、阿里云 Qwen、火山云豆包 等国产大模型均兼容 OpenAI API 格式，注册简单、支持支付宝/微信付款、无需海外信用卡，强烈推荐国内用户优先选择。

智谱 GLM 国产推荐

GLM-5.1 系列模型。国内注册，支付宝付款，支持中英双语，价格便宜。

阿里云 Qwen 国产推荐

通义千问 Qwen-3.6 系列。阿里云百炼平台，支持支付宝，模型丰富。

火山云豆包 国产推荐

字节跳动豆包系列模型。火山引擎平台，支持支付宝，响应速度快。

DeepSeek 国产推荐

DeepSeek-V3/R1 系列。推理能力极强，价格极低，注册简单。

OpenRouter

聚合多个模型，按量付费，$5 即可开始。支持 GPT-4o、Claude、Gemini 等。

OpenAI

ChatGPT 官方 API。需要海外信用卡，门槛较高。

Anthropic

Claude 官方 API。需要海外信用卡。对话质量高。

Ollama（本地免费）

完全免费，无需网络。需要 8GB+ 内存。

方案 A：申请智谱 GLM API Key（国产推荐）

1注册智谱开放平台账号
  • 访问 https://open.bigmodel.cn/
  • 点击右上角 "注册"，使用手机号注册（国内手机号即可）
  • 完成实名认证（需上传身份证，一般几分钟审核通过）
2获取 API Key
  • 登录后进入 API Keys 管理页面
  • 点击 "添加新 API Key"
  • 复制生成的 Key（格式：xxxxxxxx.xxxxxxxx）立即保存，关闭窗口后无法再查看
3充值余额
  • 进入 账户充值页面
  • 支持 支付宝、微信支付
  • 新注册用户通常赠送免费额度，可直接体验

可用模型

GLM-5.1-Plus（推荐，综合能力强）、GLM-5.1-Air（轻量，响应快）、GLM-5.1-Flash（免费额度）

方案 B：申请阿里云 Qwen API Key（国产推荐）

1注册阿里云百炼平台
  • 访问 https://bailian.console.aliyun.com/
  • 使用阿里云账号登录（淘宝/支付宝账号可直接登录）
  • 首次使用需开通"百炼"服务（免费开通）
2获取 API Key
  • 进入百炼控制台后，点击右上角头像 → "API-KEY 管理"
  • 点击 "创建新的 API Key"
  • 复制生成的 Key（格式：sk-xxxxxxxxxxxxxxxx）立即保存
3充值余额
  • 在百炼控制台 → "资源包管理" 或 "账户中心"
  • 支持 支付宝 充值
  • 新用户通常赠送 100 万+ 免费 token

可用模型

qwen-plus（推荐，性价比高）、qwen-turbo（轻量快速）、qwen-max（旗舰模型）、qwen-long（超长上下文）

方案 C：申请火山云豆包 API Key（国产推荐）

1注册火山引擎账号
  • 访问 https://www.volcengine.com/
  • 点击 "注册"，使用手机号注册
  • 完成企业/个人实名认证
2开通豆包大模型服务
  • 进入 火山方舟控制台
  • 选择 "模型推理" → 创建推理接入点
  • 选择模型（如 Doubao-1.5-pro）
3获取 API Key
  • 进入 API Key 管理
  • 点击 "创建新的 API Key"
  • 复制 Key（格式：xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx）
4获取接入点 Endpoint ID
  • 在推理接入点列表中，找到你创建的接入点
  • 复制 Endpoint ID（格式：ep-xxxxxxxx），配置模型时需要用到

可用模型

Doubao-1.5-pro（推荐）、Doubao-1.5-lite（轻量）、Doubao-1.5-32k（长上下文）

方案 D：申请 DeepSeek API Key（国产推荐）

1注册 DeepSeek 账号
  • 访问 https://platform.deepseek.com/
  • 使用手机号或邮箱注册
2获取 API Key
  • 进入 API Keys 页面
  • 点击 "创建 API Key"
  • 复制 Key（格式：sk-xxxxxxxxxxxxxxxx）

可用模型

deepseek-chat（即 DeepSeek-V3，推荐）、deepseek-reasoner（即 DeepSeek-R1，推理增强）

方案 E：申请 OpenRouter API Key（海外）

1注册 OpenRouter 账号
  1. 访问 https://openrouter.ai/
  2. 点击右上角 "Sign In"
  3. 使用 Google 账号或 GitHub 账号登录
2创建 API Key
  1. 登录后，点击右上角头像 → "Keys"
  2. 点击 "Create Key"
  3. 给 Key 起个名字（如 openhuman），点击创建
  4. 立即复制保存 Key（格式：sk-or-v1-xxxxx...），页面关闭后无法再查看
3充值余额
  1. 点击头像 → "Credits"
  2. 点击 "Add Credits"
  3. 最低充值 $5 即可使用

每条消息费用约 $0.001-0.03，$5 大约可用数千条消息。

方案 F：申请 OpenAI API Key（海外）

1注册并创建 Key
  1. 访问 https://platform.openai.com/
  2. 注册 → 进入 API Keys 页面
  3. 点击 "Create new secret key" → 复制保存（sk-xxxxx...）
2充值
  1. 访问 Billing 页面
  2. 绑定海外信用卡，充值至少 $5

OpenAI 需要海外信用卡（Visa/Mastercard）。中国大陆信用卡可能被拒绝，建议用 OpenRouter 替代。

方案 G：申请 Anthropic API Key（海外）

  1. 访问 https://console.anthropic.com/ 注册
  2. 进入 API Keys 页面 → "Create Key"
  3. 复制保存（sk-ant-xxxxx...）

方案 H：使用 Ollama 本地模型（完全免费）

完全免费，无需 API Key！

Ollama 在你自己的电脑上运行 AI 模型，数据不离开你的机器。

1安装 Ollama
  1. 访问 https://ollama.com/
  2. 点击 "Download"，选择对应平台下载安装
  3. 安装完成后，Ollama 会自动在后台运行
2下载 AI 模型
# 下载 Llama 3（约 4.7 GB，推荐 8GB+ 内存）

# 或者更小的模型（约 2.3 GB，适合 8GB 内存）
ollama pull llama3:8b

# 下载中文能力更好的模型

模型下载需要网络。下载完成后可离线使用。

验证 Ollama 运行
ollama list

应列出你下载的模型。

API Key 安全说明

API Key 不会存储在 .env 文件中。它们通过应用内设置输入后，使用 AES-256-GCM 加密存储在 auth-profiles.json 中，密钥通过 Argon2id 从用户密码派生。

第十三章：常见问题 FAQ

Q1：CEF 下载失败怎么办？▼

CEF 约 400MB，网络问题可能导致失败：

  1. 确保代理已配置（第三章）
  2. 设置环境变量：export https_proxy=http://127.0.0.1:7890
  3. 重试：pnpm dev:app（CEF 有缓存，不会重复下载）
Q2：Node.js 版本不对？▼
fnm install 24
fnm use 24
fnm default 24
Q3：Windows 报 MSVC 错误？▼

安装 Visual Studio Build Tools，勾选 "Desktop development with C++"，重启终端。

Q4：如何完全免费使用？▼
  1. 安装 Ollama：ollama.com
  2. 下载模型：ollama pull llama3
  3. 在 OpenHuman 中选择"自定义模式" → 启用 Ollama → 选择 ollama:llama3
  4. 工作负载路由全部选 ollama 模型即可
Q5：应用白屏？▼
  1. 检查终端是否有错误输出
  2. 检查 .env 配置
  3. 清除缓存：rm -rf node_modules/.vite 后重试
  4. 检查端口占用
Q6：如何只下载安装包？▼

访问 GitHub Releases 下载对应平台安装包，安装后参考第五、六章配置 AI 即可。

Q7：Ollama 下载模型慢？▼
export http_proxy=http://127.0.0.1:7890
export https_proxy=http://127.0.0.1:7890
ollama pull llama3
Q8：如何查看日志？▼
pnpm debug logs         # 列出最近日志
pnpm debug logs last    # 查看最近日志
Q9：国产模型报 401/403 错误？▼
  1. 确认 API Key 正确且未过期
  2. 确认账户余额充足（部分平台余额为 0 也会报 403）
  3. 确认 Endpoint URL 正确（阿里云必须用 /compatible-mode/v1 路径）
  4. 确认 Auth Style 选了 Bearer（国产模型统一用 Bearer）
Q10：火山云豆包模型填什么名字？▼

火山云豆包比较特殊，模型名不填模型本身的名字，而是填你在方舟控制台创建的 "接入点 ID"（Endpoint ID）。

  1. 进入 火山方舟控制台
  2. 创建推理接入点 → 选择模型（如 Doubao-1.5-pro）
  3. 在接入点列表中复制 ID（格式：ep-20250524xxxxx）
  4. 在 OpenHuman 中填入：doubao:ep-20250524xxxxx
Q11：可以用国产模型 + 海外模型混合吗？▼

可以！OpenHuman 支持按工作负载分别配置不同提供商。例如：

  • chat → zhipu:glm-5.1-plus（日常对话用国产模型，便宜）
  • reasoning → deepseek:deepseek-reasoner（深度推理用 DeepSeek R1）
  • embeddings → ollama:bge-m3（嵌入用本地模型，免费）

你可以把便宜的国产模型用在大部分场景，只在需要高端推理时切换到更强的模型。

Q12：OpenHuman 是完全免费的吗？为什么要登录？▼
  • 软件本身：完全免费开源（GPL v3）
  • 使用 OpenHuman 托管 AI：需要注册登录，按 credits 收费（有免费额度）
  • 使用自己的 API Key（国产模型/海外模型）：不产生 OpenHuman 平台费用，费用付给对应的模型服务商
  • 使用本地 Ollama：完全免费，无需登录

所以"需要登录和收费"指的是 OpenHuman 托管的 AI 服务。如果你自己配了智谱 GLM 或本地 Ollama，就绕过了这套计费。
