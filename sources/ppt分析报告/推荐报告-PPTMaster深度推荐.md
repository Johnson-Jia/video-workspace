## 一、为什么 PPT Master 值得关注

一句话概括：**它是目前所有开源 AI PPT 项目中，输出质量天花板最高的一个。**

其他项目（Presenton、ALLWEONE、PPTAgent）生成的 PPT，本质上都是"HTML 转换"或"图片拼接"——看起来像 PPT，但打开 PowerPoint 编辑时要么元素乱飞，要么文字变成不可编辑的图片块。

PPT Master 走了一条完全不同的路：**生成原生 PowerPoint 元素**。每个文字框、每个形状、每张图片、每个图表，都是 PowerPoint 能直接识别和编辑的原生对象。这意味着生成的 PPT 不是"终稿快照"，而是一个**真正可以继续修改的工作文件**。

这个差异在实际使用中极其关键——没有哪个 AI 能一次生成完美 PPT，用户拿到后总要微调。如果元素不可编辑，"微调"就等于重做。

## 二、核心能力深度解析

### 2.1 原生可编辑 PPTX — 行业唯一

**技术原理：SVG → DrawingML 转换管线**

PPT Master 不直接生成 PPTX，而是先让 AI 生成 SVG（矢量图形），再通过 Python 转换引擎（`svg_to_pptx.py`）将 SVG 转换为 PowerPoint 的 DrawingML 格式。

为什么选 SVG 作为中间格式？项目在 33000 字的技术设计文档（`docs/technical-design.md`）中给出了严格的排除法论证：

| 候选格式 | 排除原因 |
|---------|---------|
| HTML/CSS | 无法精确控制 PowerPoint 的元素定位 |
| python-pptx 直接生成 | AI 对 DrawingML API 的理解成本太高，容易出错 |
| LaTeX Beamer | 样式能力有限，无法实现现代设计 |
| **SVG** ✅ | AI 擅长生成、坐标精确、矢量无损、可转 DrawingML |

**实际效果：**
- 文字框 → PowerPoint 原生 TextBox，可双击编辑文字、改字体、调大小
- 形状 → PowerPoint 原生 Shape，可改颜色、调位置、加动画
- 图片 → PowerPoint 原生 Picture，可替换、裁剪、加边框
- 图表 → PowerPoint 原生 Chart，可编辑数据、改图表类型
- 动画 → PowerPoint 原生 Animation，可在动画窗格中逐帧调整

**品牌系统（`create-brand` 工作流）：**
- 定义品牌色、品牌字体、Logo 规范
- 一次创建，所有页面自动遵循品牌识别体系
- 支持导出和复用品牌配置

### 2.6 质量保障体系

| 工具 | 功能 |
|------|------|
| `svg_quality_checker.py` | SVG 质量检查，检测渲染错误 |

## 三、架构设计亮点

### 3.1 Harness 范式 — 创新的工作流设计

PPT Master 不是传统意义的"应用程序"，而是一个 **harness（工具框架）**：

```
harness（本仓库）+ model（AI IDE）= agent（PPT 生成器）
```

这种设计的优势：
- **不绑定特定 AI 模型**：Claude、GPT、Gemini 都能用
- **不绑定特定界面**：Claude Code、Cursor、Trae、Windsurf 都能驱动
- **工作流即文档**：SKILL.md 就是工作流定义，人可读、AI 可执行
- **用户全程可控**：每一步都可以审查和修改，不是"黑箱一键生成"

### 3.2 三角色协作管线

```
Strategist（策略师）
  ├── 分析源文档内容
  ├── 确定设计风格和色调
  ├── 规划每页内容结构
  └── 八步确认流程（用户可审核每一步）

Image_Generator（图片生成器）
  ├── 根据策略批量生成配图
  └── Manifest 模式保证风格统一

Executor（执行器）
  ├── 逐页生成 SVG
  ├── 实时预览
  └── 质量检查 → 修正 → 导出 PPTX
```

### 3.3 33000 字技术设计文档

`docs/technical-design.md` 是所有分析项目中**文档质量最高**的：
- 每个架构决策都有"为什么这么选"的论证过程
- 包含完整的格式对比排除法（HTML vs SVG vs LaTeX vs python-pptx）
- 包含上下文管理策略（spec_lock 反漂移机制）
- 包含错误处理和边界情况的分析

## 五、实际场景深度验证

### 场景 A：产品经理做产品发布会 PPT（20 页）

**输入：** 一份产品 PRD 文档（Word 格式，5000 字）

**操作流程：**
1. 在 Claude Code 中输入 `/ppt-master`
2. 导入 PRD 文档 → 自动转为 Markdown
3. 选择"毛玻璃 SaaS 风"模板
4. Strategist 分析文档，8 步确认内容策略（约 3 分钟）
5. Image_Generator 批量生成产品配图（约 2 分钟）
6. Executor 逐页生成 SVG + 实时预览（约 5 分钟）
7. 质量检查 → 导出 PPTX

**总耗时：** 约 15-20 分钟
**输出质量：** ⭐⭐⭐⭐⭐ — 每页都是原生可编辑元素，动画、图表、排版全部专业级
**后续调整：** 在 PowerPoint 中直接修改，和手动做的 PPT 体验完全一致

**对比其他工具：**
- Presenton/ALLWEONE：5 分钟出 PPT，但元素不可编辑，后续调整困难
- PPT Master：15 分钟出 PPT，但质量极高且完全可编辑，后续调整无障碍

### 场景 B：研究生做论文答辩 PPT（15 页）

**输入：** 一篇已发表的学术论文（PDF 格式）

**操作流程：**
1. 导入 PDF → `pdf_to_md.py` 自动提取内容
2. 选择"瑞士风"模板（学术严谨感）
3. Strategist 提取核心论点、实验数据、结论
4. Executor 生成包含数据图表的答辩幻灯片

**输出质量：** ⭐⭐⭐⭐ — 排版专业、数据清晰，但复杂公式可能需要手动调整
**优势：** 从论文到答辩 PPT 的内容提取非常精准，不会遗漏关键论点

### 场景 C：创业公司做融资 BP（12 页）

**输入：** 业务计划书（Word）+ 财务数据（Excel）

**操作流程：**
1. 同时导入 Word + Excel → 自动合并为 Markdown
2. 选择"财经数据风"模板
3. Strategist 规划：市场分析 → 产品介绍 → 商业模式 → 财务预测 → 团队介绍
4. Image_Generator 生成行业数据可视化配图
5. Executor 生成包含数据仪表盘的幻灯片

**输出质量：** ⭐⭐⭐⭐⭐ — 数据仪表盘 + 专业排版，投资人看到不会觉得是 AI 生成的
**差异化：** 这是唯一能把 Excel 财务数据直接转化为 PPT 数据图表的工具

## 八、快速上手指南（给会用 AI IDE 的用户）

### Step 1：安装 AI IDE

选择以下之一：
- **Claude Code**（推荐）：`npm install -g @anthropic-ai/claude-code`
- **Cursor**：下载 https://cursor.com
- **Trae**：下载 https://trae.ai

### Step 2：克隆 PPT Master

```bash
git clone https://github.com/hugohe3/ppt-master.git
cd ppt-master
```

### Step 3：在 AI IDE 中使用

在 Claude Code 中：
```
请阅读 skills/ppt-master/SKILL.md，然后帮我基于 projects/ 目录创建一个新项目
```

### Step 4：导入素材 → 生成 PPT

把你的文档（PDF/Word/Excel）放入项目目录，AI 会引导你完成：
1. 文档转换 → 2. 模板选择 → 3. 策略确认 → 4. 图片生成 → 5. SVG 制作 → 6. 质量检查 → 7. 导出 PPTX
