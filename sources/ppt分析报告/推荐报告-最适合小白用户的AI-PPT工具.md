# 5 款 AI PPT 开源项目全方位对比与推荐

> 基于源码深度分析的实据推荐，目标：找出最好用、最易安装、让普通人和小白用户都能简单制作精美 PPT 的工具

---

## 一、五维度淘汰赛

先根据用户的核心诉求——"好用、易用、易安装、小白友好、输出精美"——做五轮淘汰。

### 第 1 轮：小白能不能装上？

| 项目 | 安装方式 | 最低门槛 | 淘汰判定 |
|------|---------|---------|---------|
| **Presenton** | 下载 .exe/.dmg 安装包，双击即用 | 会下载软件就行 | **通过** |
| **ALLWEONE** | 在线 Demo 可直接用（presentation.allweone.com） | 有浏览器就行 | **通过** |
| **ai-to-pptx** | 本地部署需 PHP + Node.js 双后端；在线版依赖作者远程服务器 | 本地部署不可能；在线版数据不安全 | **淘汰** — 本地部署不可能，在线版核心逻辑不可控、数据发到第三方服务器 |
| **PPT Master** | 必须安装 AI IDE（Claude Code / Cursor / Trae）+ Python + API Key | 需要理解 AI IDE 概念 | **淘汰** — 普通人根本不知道什么是 AI IDE |
| **PPTAgent** | 需要 Python 3.11+ + Docker + WSL（不支持 Windows）+ 多个 API Key | 需要命令行操作、Docker 知识 | **淘汰** — Windows 用户直接出局 |

**第 1 轮淘汰：PPT Master、PPTAgent、ai-to-pptx**

一个工具再强大，装不上就是零。这三个项目对普通用户的安装门槛分别是：
- PPT Master：你得先学会用 Claude Code 或 Cursor（这是开发者的工具），然后还要有 Claude API Key（要注册国外服务、绑定信用卡）
- PPTAgent：Windows 原生不支持，得装 WSL（Windows 子系统 Linux），再装 Docker，再配置 3-4 个 API Key
- ai-to-pptx：前后端分离架构，后端是 PHP，得同时跑两个服务；在线版虽然能直接用，但你的 PPT 内容全部发送到作者的私人服务器 `fdzz.dandian.net:8443`，商业文件直接泄露

### 第 2 轮：装上之后会不会用？

| 项目 | 交互方式 | 操作步骤 | 学习成本 |
|------|---------|---------|---------|
| **Presenton** | 桌面应用 / Web 界面 | 输入主题 → AI 生成大纲 → 确认 → 生成 PPT → 导出 | 5 分钟学会 |
| **ALLWEONE** | Web 浏览器 | 输入主题 → AI 生成大纲 → 编辑大纲 → 选主题 → 生成 → 对话编辑 | 5 分钟学会 |

**两个都通过。** 操作逻辑和 Gamma、美图 AI PPT 等商业产品一致，任何人都能理解"输入主题 → 出 PPT"这个流程。

### 第 3 轮：生成的 PPT 够不够精美？

| 项目 | 主题/模板数量 | 设计质量 | 输出格式 |
|------|-------------|---------|---------|
| **Presenton** | 内置多种模板 + 支持导入现有 PPT 作为模板 | 专业级，支持自定义配色和布局 | PPTX（可编辑）+ PDF |
| **ALLWEONE** | **38 个内置主题** + 自定义主题 + PPTX 主题导入 | 风格最丰富（daktilo、cornflower、orbit、piano、mystique 等），覆盖商务/科技/教育/休闲 | PPTX + 在线演示 + 分享链接 |

**两个都精美，但风格不同：**
- Presenton 走的是"干净专业"路线，类似 Apple Keynote 的简约感
- ALLWEONE 走的是"丰富多样"路线，38 个主题覆盖各种场景，从正式商务到创意设计都有

### 第 4 轮：生成之后能不能改？

| 项目 | 编辑能力 | 编辑方式 |
|------|---------|---------|
| **Presenton** | 对话式编辑（chat endpoint） | 用自然语言告诉 AI "把第三页的标题改成XX"、"换一个更商务的配色" |
| **ALLWEONE** | **Plate.js 专业富文本编辑器** + 对话式 AI 编辑 | 可视化拖拽、直接修改文字/图片/图表 + AI 对话编辑 |

ALLWEONE 的编辑能力更强——它有一个完整的幻灯片编辑器（基于 Plate.js），支持：
- 直接拖拽调整幻灯片顺序
- 点击修改文字内容
- 替换图片
- 切换主题（一键换装）
- AI 对话编辑（"帮我加一页总结"、"把这页改成数据图表"）

## 二、最终推荐

### 🥇 第 1 推荐：Presenton

**推荐理由：最接近"开箱即用"的 AI PPT 工具**

| 维度 | 评价 |
|------|------|
| 安装 | 下载桌面应用（.exe / .dmg），双击安装，和安装微信一样简单 |
| 上手 | 输入主题就能生成 PPT，5 分钟学会 |
| 精美度 | 专业级设计，支持自定义模板 |
| 隐私 | 支持完全本地运行（Ollama 模式），数据不出电脑 |
| AI 灵活度 | 支持 OpenAI / Gemini / Anthropic / DeepSeek / Ollama，用什么模型自己选 |
| 部署方式 | 桌面应用 / Docker / Web 三选一，覆盖所有场景 |
| 输出 | PPTX（可编辑）+ PDF |

**适合谁：**
- 完全不会写代码的普通人
- 需要快速做 PPT 的职场人
- 注重数据隐私的企业用户
- 想要离线使用的用户

**不足：**
- 主题/模板数量没有 ALLWEONE 多
- 对话编辑能力不如 ALLWEONE 的可视化编辑器

**GitHub 地址：** https://github.com/presenton/presenton

### 🥈 第 2 推荐：ALLWEONE Presentation AI

**推荐理由：功能最全、主题最丰富、编辑能力最强**

| 维度 | 评价 |
|------|------|
| 安装 | 有在线 Demo，直接用浏览器访问，无需安装任何东西 |
| 上手 | 大纲优先 → 选主题 → 生成，流程清晰 |
| 精美度 | **38 个主题**，所有项目中最丰富，风格覆盖最广 |
| 编辑 | Plate.js 富编辑器 + AI 对话编辑，双模式，所有项目中最强 |
| 分享 | 内置在线演示 + 公开链接分享，协作友好 |
| 输出 | PPTX + 在线演示 + 多种格式（16:9/4:3/1:1/9:16） |

**适合谁：**
- 想要在线直接用、不想装任何东西的用户
- 对 PPT 设计风格有要求、想要多种主题选择的用户
- 需要分享和协作的团队
- 想在浏览器里直接编辑 PPT 的用户

**不足：**
- 本地部署需要 PostgreSQL + Node.js + 多个 API Key（比 Presenton 复杂）
- PPTX 导出使用 pptxgenjs 转换，不如原生操作精确
- 不支持公式渲染（学术场景偏弱）

**在线 Demo：** http://presentation.allweone.com
**GitHub 地址：** https://github.com/allweonedev/presentation-ai

## 三、为什么不推荐另外三个

### ❌ PPT Master（8.3 分但小白评分 2/10）

技术最强、输出质量最高——但代价是你得先学会用 Claude Code 或 Cursor（这是程序员才用的工具）。它不是给普通人做的，是给懂 AI IDE 的技术用户做的。它的安装和使用流程是：

1. 安装 Claude Code（需要命令行操作）
2. 配置 Claude API Key（需要注册 Anthropic 账号、绑信用卡）
3. 在 AI IDE 中输入命令让 AI 执行工作流
4. 理解 harness、SKILL.md、Strategist 八步确认等概念

这不是"普通人"能完成的操作。

### ❌ PPTAgent（学术最强但 Windows 不支持）

中科院出品、EMNLP 2025 论文，学术实力碾压全场。但：
- Windows 原生不支持，必须装 WSL
- 需要 Docker（Docker Desktop 安装 4GB+）
- 需要配置 3-4 个 API Key（OpenAI + Tavily + MinerU 等）
- 安装过程涉及命令行操作

而且只有 5 个学术模板（beamer、thu、hit 等），商务模板为零。

### ❌ ai-to-pptx（质量最低且有隐私风险）

五个项目中综合评分最低（4.5/10）：
- 后端是 PHP，核心 AI 调用逻辑在独立仓库中，前端只是个壳
- 在线版把你的 PPT 内容发送到作者的私人服务器，商业文件直接泄露
- 代码质量差（全 any 类型、无测试、无文档）
- 模板只有 28 个，且从后端随机获取，不可选择
